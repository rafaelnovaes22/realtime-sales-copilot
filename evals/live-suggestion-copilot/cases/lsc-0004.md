# Eval Case lsc-0004 — vou_pensar / cliente cansado após múltiplas objeções (edge)

## 1. Cenário
Edge case: cliente diz "vou pensar" após já ter levantado 3 objeções diferentes na mesma call. Sinal de fadiga, não de necessidade real de reflexão.

## 2. Input

```json
{
  "buffer": [
    "Cliente: [já disse 'tá caro', 'preciso falar com minha esposa', 'deixa eu pesquisar']",
    "Closer: Então, retomando — o que ainda impede você de fechar hoje?",
    "Cliente: Sim, mas deixa eu pensar mais um pouco."
  ],
  "estado": "fechamento",
  "gatilhos": ["vou_pensar"]
}
```

## 3. Gabarito (rubrica llm-as-judge)

PASSA se:
- [ ] ≤ 2 linhas / ≤ 280 chars
- [ ] Nomeia gentilmente o padrão (múltiplas objeções = algo mais fundo)
- [ ] Convida o cliente a revelar o bloqueio real — sem pressão
- [ ] Zero brand leak

**Exemplo que PASSA:** "Percebo que você levantou alguns pontos ao longo da conversa. Tem algo específico que ainda não ficou claro pra você?"

**Reprova:** Responde como se fosse a primeira objeção; pressiona; cita marca.

## 4. Justificativa
Padrão de múltiplas objeções indica objeção não declarada — medo, desconfiança profunda ou prioridade real diferente. Repetir respostas isoladas não resolve.
